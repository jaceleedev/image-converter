# 개발 문서

루트에는 사용자 진입점인 `README.md` 와 에이전트 작업 규칙인 `AGENTS.md` 만 둡니다. 상세 개발 문서는 이 디렉토리에서 관리합니다.

## 문서 목록

- `architecture.md`: 프로젝트 구조, 모듈별 책임, 의존성 흐름
- `testing.md`: Docker/로컬 테스트 실행법, 테스트 목록, 테스트 유틸리티
- `memory.md`: 최근 작업 로그, 결정 기록, 진행/대기 항목

## 문서 관리 규칙

- 사용자에게 필요한 설치/사용법은 루트 `README.md` 에 둡니다.
- 에이전트가 매 세션 따라야 하는 규칙은 루트 `AGENTS.md` 에 둡니다.
- 시간에 따라 변하는 작업 내역은 `memory.md` 에만 추가합니다.
- 코드 구조나 테스트 목록이 바뀌면 `architecture.md` 와 `testing.md` 를 함께 갱신합니다.
